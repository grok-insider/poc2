# ADR-0010 — Desktop shell: Electron windowed app, cross-platform scope (Linux, NixOS, Windows 11)

- Status: **Accepted**
- Date: 2026-06-10
- Supersedes: [ADR-0002](0002-platform-nixos-only.md) (platform scope).
- Amends: [ADR-0001](0001-tech-stack.md) (its rejection of Electron predates
  the WASM engine; see "Why Electron now").
- Leaves intact: [ADR-0009](0009-defer-wayland-layer-shell-to-v1-1.md) — the
  desktop app is a normal window, **not** an overlay; layer-shell stays deferred.

## Context

The product direction changed by explicit user decision (2026-06-10):

1. Ship a **desktop app** built from the existing web app (`apps/web`) — a
   normal windowed application like Discord, not an in-game overlay.
2. Add **item capture**: global hotkey → the game's own Ctrl+C copies the
   hovered item to the clipboard → the app imports it. This is how Awakened
   PoE Trade and Exiled Exchange 2 work (verified by reading both codebases —
   they do *not* OCR; see `example-repos/`).
3. Add **price checking** for captured items.
4. Builds and tests must work on **Linux, NixOS, and Windows 11**.

This reverses ADR-0002's "NixOS + Hyprland only" scope. ADR-0002 anticipated
this: it committed v1 to *not hardcoding Linux assumptions in ways that
prevent future cross-platform extension*, which the 2026-06 readiness audit
largely confirmed (engine and web layers are platform-free; the gaps were
`HOME`-only path resolution, a bash-only WASM build script, and Nix-only CI).

## Why Electron (and not Tauri / Electrobun / bare web)

Framework research (2026-06, see CHANGELOG and the capture research notes):

- **Tauri** stays rejected: the original WebKitGTK-under-Hyprland rendering
  failure (ADR-0001 supersede note) plus `tauri-plugin-global-shortcut` being
  X11-only and explicitly disabled on Wayland.
- **Electrobun** rejected for now: Linux support is Ubuntu-official /
  community-tier elsewhere, the artifact pipeline assumes FHS (hostile to
  NixOS), its GlobalShortcut API is X11 `XGrabKey` (dead on Wayland), and the
  default Linux renderer is WebKitGTK — the engine family we already retired.
- **Electron** is the only shell whose Wayland story is actively maintained
  (GlobalShortcuts portal support since 35, `preferred_trigger` fixes in 41)
  and whose packaging is mature on both Windows and Linux. ADR-0001 rejected
  it citing "no Rust integration" — moot since the engine now ships as WASM
  and runs in any Chromium renderer unchanged; the binary-size objection is
  accepted as the cost of the new scope.
- **Bare web app** remains fully supported: the desktop shell is additive.
  Capture/price-proxy features detect the bridge and degrade gracefully in a
  plain browser.

## Decision

1. **`apps/desktop`** hosts the Electron app (main + preload, TypeScript,
   Bun-managed like `apps/web`). The renderer is the unmodified `apps/web`
   static export, served over a privileged custom scheme (`app://`) so the
   export's root-absolute asset paths work without a localhost server.
2. **Renderer contract**: the preload exposes exactly one bridge,
   `window.poc2Desktop` (typed in `apps/web/lib/desktop.ts`). The web app
   never imports Electron; the desktop app never reaches into web internals.
3. **Capture** lives in the Electron main process behind one interface with
   per-platform backends:
   - Windows: `globalShortcut` + key injection via `uiohook-napi`
     (optionalDependency; loaded lazily, Windows-only).
   - Linux/Wayland: GlobalShortcuts portal where available, plus a documented
     compositor-bind fallback (`hyprctl dispatch sendshortcut`, then
     `ydotool`) — spawned tools, no native Node modules on Linux.
   - Clipboard flow copies APT semantics: snapshot → inject Ctrl+C → poll
     until `Item Class:` prefix → restore clipboard.
4. **Price checking**: trade2 API (`POST /api/trade2/search/{league}`,
   `GET /api/trade2/fetch/...`) is called from the **main process** (no CORS,
   centralized rate limiting per the API's `X-Rate-Limit-*` headers). The
   mod→trade-stat-id table is generated by the pipeline into the bundle
   (source: official `/api/trade2/data/stats`, cross-checked against the
   MIT-licensed Exiled-Exchange-2 dataset in `example-repos/`). Browser-only
   sessions fall back to trade deep links + poe2scout estimates.
5. **Platform branches** (`process.platform`, `cfg(target_os)`) are allowed
   **only** in `apps/desktop` and explicitly-marked operator tooling. Engine,
   data, advisor, parser, market, wasm, and web layers stay platform-free.
6. **CI** gains a `windows-latest` lane (rustup + Bun, no Nix) building and
   testing the workspace, the WASM artifact, the web export, and the Electron
   package. Nix lanes remain canonical for Linux/NixOS.
7. **Distribution**: dev runs use nixpkgs `electron` on NixOS;
   `electron-builder` produces AppImage/deb (Linux) and NSIS (Windows 11) in
   CI. macOS stays out of scope.

## Consequences

- ADR-0002's "politely close non-NixOS bug reports" policy ends. Windows 11
  is a supported target; macOS is not.
- The capture path reintroduces deliberate, *isolated* native coupling —
  exactly the per-platform-backend shape ADR-0002 demanded.
- The web app gains a hard rule: all runtime asset URLs must stay
  origin-relative (they are served via `app://` in the desktop shell).
- AGENTS.md is the living source for the conventions this ADR fixes.
